Desplegar el stack

Iniciar sesión y elegir la región

Ingresá a la consola de AWS y seleccioná la región del laboratorio.

Consola AWS› selector de región (arriba a la derecha) › US East (N. Virginia) us-east-1

Abrir CloudFormation y crear el stack

CloudFormation› Stacks › Create stack › With new resources

Subí el archivo template.yaml de esta carpeta en Upload a template file.

Nombre y parámetros

Poné un nombre al stack (por ejemplo obs-clase-N) y revisá los parámetros.

Alternativa por CLISi preferís la línea de comandos:
bash · desplegar por CLI
aws cloudformation deploy \
  --stack-name obs-clase-N \
  --template-file template.yaml \
  --capabilities CAPABILITY_NAMED_IAM \
  --region us-east-1

Confirmar y crear

Aceptá las capacidades (IAM) y presioná Submit. Esperá el estado CREATE_COMPLETE.

Checkpoint
El stack aparece en estado CREATE_COMPLETE y la pestaña Resources lista los recursos creados.

Troubleshooting

Problemas frecuentes y cómo resolverlos. Ideal para practicar diagnóstico.

El stack queda en ROLLBACK_COMPLETE

Síntoma: la creación falla y el stack se revierte.

Solución: abrí la pestaña Events, buscá el primer error (arriba del rollback) y corregí el parámetro o permiso indicado.

No aparecen métricas en CloudWatch

Síntoma: el namespace está vacío.

Solución: esperá 3–5 minutos; verificá el rol/permisos de la instancia y que el agente esté corriendo.

Limpieza

Eliminar el stack

CloudFormation› Stacks › obs-clase-N › Delete
bash · eliminar por CLI
aws cloudformation delete-stack --stack-name obs-clase-N --region us-east-1
Checkpoint
El stack desaparece de la lista (estado DELETE_COMPLETE). No quedan recursos facturables.
