Monitoreo, observabilidad y sus pilares

Monitoreo responde a preguntas que ya sabías hacer (¿está caído?, ¿pasó el umbral?). La observabilidad te permite explorar el sistema y responder preguntas nuevas que no anticipaste.

Los tres pilares (+ eventos)

  • Métricas: valores numéricos agregados en el tiempo.
  • Logs: registros discretos de lo que ocurrió.
  • Trazas: el recorrido de una petición entre servicios.
  • Eventos: cambios de estado relevantes del sistema.

Módulo 1 · Qué medir

Señales, métodos y objetivos de servicio

Golden signals

Latencia, tráfico, errores y saturación: las cuatro señales que resumen la salud de un servicio orientado a usuarios.

RED & USE

RED (Rate, Errors, Duration) para servicios de peticiones; USE (Utilization, Saturation, Errors) para recursos como CPU o disco.

SLI · SLO · Error budget

El SLI mide, el SLO fija la meta y el error budget es cuánto podés fallar sin romper el objetivo.

🧭 Idea clave

Estos marcos deciden qué métricas valen la pena. CloudWatch es el dónde las recolectás y consultás.

Métricas con Amazon CloudWatch

Cómo se organizan, se publican y se consultan las métricas en la plataforma central de observabilidad de AWS.

Observabilidad y Monitoreo en AWS ·Clase 1
Management & Governance

Amazon CloudWatch

Qué es

Servicio de observabilidad que recolecta métricas, logs y eventos de recursos y aplicaciones. Provee métricas nativas de AWS y permite publicar las tuyas, con alarmas, dashboards y consultas.

Conceptos clave

  • Namespaces y dimensiones: namespace (AWS/EC2, CWAgent) + nombre + dimensiones (InstanceId) definen una métrica única.
  • Resolución y retención: estándar (1 min) vs alta (hasta 1 s); se conservan hasta 15 meses con granularidad decreciente.

💡 En este curso

Es el destino de todas las métricas del lab: las del hipervisor de EC2 (AWS/EC2), las del SO que publica el agente (CWAgent) y la métrica personalizada.

Se usa en el lab

Módulo 2 · Publicar y consultar

De PutMetricData a Metrics Insights

  • PutMetricData: API para publicar métricas personalizadas (valores, arrays o StatisticSet).
  • EMF: métricas embebidas en logs JSON que CloudWatch extrae, sin llamadas síncronas.
  • Metric math combina y transforma métricas; la detección de anomalías aprende una banda esperada.
  • Metrics Insights: consultas tipo SQL sobre tus métricas más recientes.
Buena práctica

Cuidá la cardinalidad: cada combinación de dimensiones es una métrica facturable. EMF ayuda a controlar el volumen.

# publicar una métrica personalizada
aws cloudwatch put-metric-data \
  --namespace "Curso/Obs" \
  --metric-name "TareasProcesadas" \
  --dimensions InstanceId=i-0abc123 \
  --value 1

# consulta tipo SQL (Metrics Insights)
SELECT AVG(mem_used_percent)
FROM "CWAgent"
GROUP BY InstanceId

Monitoreo de EC2 con el agente de CloudWatch

El hipervisor no ve dentro del sistema operativo. El agente unificado sí: memoria, disco, procesos y logs.

Observabilidad y Monitoreo en AWS ·Clase 1
Compute

Amazon EC2 y el agente unificado

Qué es

EC2 provee instancias de cómputo y publica métricas del hipervisor sin agente. Pero no ve lo que pasa dentro del SO invitado.

Conceptos clave

  • Sin agente: CPUUtilization, NetworkIn/Out, StatusCheckFailed en AWS/EC2.
  • El hipervisor no ve memoria usada, uso de disco por filesystem ni procesos: eso requiere el agente corriendo dentro.
  • UserData instala y configura el agente en el primer arranque.

💡 En este curso

Una t3.micro es el host monitoreado. Su UserData instala el agente, que lee su config desde SSM y publica mem_used_percent y disk_used_percent en CWAgent.

Se usa en el lab

Módulo 3 · Config y despliegue

Config versionada y acceso sin SSH

AWS Systems Manager
  • Parameter Store guarda la config JSON del agente de forma versionada; la instancia la recupera al arrancar.
  • Session Manager te da shell a la instancia sin SSH, sin puertos abiertos ni bastión.
  • Run Command despliega el agente a escala en flotas de instancias.
AWS IAM
  • El agente accede a AWS mediante un instance profile, sin credenciales estáticas.
  • CloudWatchAgentServerPolicy habilita PutMetricData y leer la config en SSM.
  • AmazonSSMManagedInstanceCore habilita Session Manager.
⚠️ Error frecuente

Si buscás mem_used_percent en AWS/EC2 no aparece: vive en CWAgent y solo existe si el agente está corriendo y configurado.

Módulo 4 · Infraestructura como código

AWS CloudFormation

Anatomía de una plantilla

  • Parameters: entradas configurables del stack.
  • Resources: la única sección obligatoria; declara qué crear.
  • Outputs: valores exportables (IDs, endpoints, comandos).
  • Funciones intrínsecas: !Ref, !GetAtt, !Sub componen valores entre recursos.

💡 Por qué como código

Definir la observabilidad como código (VPC, EC2, IAM, config del agente) da entornos reproducibles, versionables y auditables. Un change set previsualiza los cambios.
