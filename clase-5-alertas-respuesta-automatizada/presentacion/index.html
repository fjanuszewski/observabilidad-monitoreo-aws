Alarmas de CloudWatch

Qué es

Una alarma observa una métrica (o una expresión de métricas) contra un umbral durante un número de periodos y cambia de estado para disparar acciones.

Tres tipos

  • De métrica: umbral estático sobre una métrica, p. ej. CPUUtilization > 80%.
  • Compuesta: combina varias alarmas con lógica AND/OR para reducir ruido.
  • De anomalía: aprende una banda esperada y alerta cuando la métrica se sale de ella.

💡 En este curso

El lab crea las tres: una de métrica sobre la CPU, una compuesta que agrupa señales y una por detección de anomalía. La alarma de CPU dispara la auto-remediación.

Módulo 1 · Estados y acciones

Estados, datos faltantes y qué disparan

Estados

  • OK: la métrica está dentro del umbral.
  • ALARM: se cruzó el umbral el tiempo definido.
  • INSUFFICIENT_DATA: faltan datos para evaluar.

Datos faltantes

  • notBreaching, breaching, ignore o missing: definen cómo tratar los periodos sin dato.

⚙️ Acciones de alarma

  • SNS: notifica a personas o sistemas.
  • Auto Scaling: escala en respuesta a la carga.
  • Acciones de EC2: reiniciar, detener o recuperar.
  • Systems Manager: abrir un OpsItem o disparar automatización.

Amazon EventBridge

Qué es

El bus de eventos serverless de AWS (antes CloudWatch Events). Enruta eventos de servicios de AWS y de tus aplicaciones hacia destinos, según patrones.

Conceptos clave

  • Buses: el bus por defecto recibe eventos de AWS; podés crear buses propios.
  • Reglas y patrones: un patrón JSON filtra eventos (origen, tipo, detalle).
  • Destinos: Lambda, SSM Automation, SNS, Step Functions y más.

💡 En este curso

Una regla escucha el cambio de estado de la alarma de CPU a ALARM y dispara el runbook de SSM que remedia el incidente automáticamente.

Módulo 2 · Remediar y avisar

Auto-remediación y notificaciones

SSM Automation · Lambda
  • Runbooks de SSM Automation: pasos declarativos y reutilizables para remediar (reiniciar un servicio, aislar una instancia).
  • Lambda: lógica de remediación a medida cuando el runbook no alcanza.
SNS · Chatbot
  • SNS fan-out hacia email, Slack o PagerDuty.
  • AWS Chatbot lleva las alertas a canales de Slack o Teams.
# patrón de evento: alarma que pasa a ALARM
{
  "source": ["aws.cloudwatch"],
  "detail-type": ["CloudWatch Alarm State Change"],
  "detail": {
    "state": { "value": ["ALARM"] }
  }
}
Buena práctica

Que la remediación sea idempotente y deje rastro: registrá cada ejecución del runbook para poder auditar qué se automatizó.

Estrategia de monitoreo para producción

Alertar para actuar, no para saturar: umbrales accionables, SLIs/SLOs, runbooks y detección temprana.

Observabilidad y Monitoreo en AWS ·Clase 5
Módulo 4 · Alertas que valen la pena

Umbrales accionables y menos ruido

Reducir el ruido

Alertá solo sobre síntomas accionables. Usá alarmas compuestas y umbrales por duración para evitar la fatiga de alertas.

SLIs y SLOs

El SLI mide la experiencia; el SLO fija la meta. Alertá cuando el error budget se agota, no ante cada pico.

Runbooks operativos

Cada alerta apunta a un runbook: qué significa, cómo diagnosticar y cómo remediar. La detección temprana acorta el impacto.

🧭 Idea clave

Si una alerta no requiere una acción humana o automática, no debería despertar a nadie: mandala a un dashboard, no a la guardia.
